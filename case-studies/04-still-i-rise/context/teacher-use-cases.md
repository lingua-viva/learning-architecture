**Status**: `observed` — real teacher workflows from IB classroom experience
**Source**: Claudia Canu Fautré, curriculum architect, 6 years IB international school

## Use Case 1: Per-Student Lens (Differentiated Content)

**The scenario**: A teacher is planning tomorrow's lesson on multiplication. One student has dyslexia. Another's Italian is still low. The textbook is too complicated for some (missing articles, complex sentence structure).

**What the teacher needs**: Customized exercises for THAT student — same concept (multiplication), adapted to their profile (reading level, learning difference, language proficiency).

**What MC would do**:
1. Teacher describes the student's situation (speech-to-text → structured note)
2. MC loads the student lens (learning profile, CEFR level, documented needs)
3. MC generates differentiated content: same mathematical concept, adapted language level, visual supports if flagged, shorter tasks if needed
4. Teacher reviews, adjusts, delivers

**Key constraint**: Feedback collection is VERY difficult to do regularly. The system must make observation capture trivially easy — speech-to-text to notepad, then upload to lens. Not forms. Not rubrics. Talk and tag.

## Use Case 2: Tiered Intervention (3 Tiers)

**Tier 1 — Universal**: Standard instruction. All students receive this.

**Tier 2 — Targeted**: Student needs specific attention but no diagnosis. Work with Italian support team. Possible needs:
- Visual supports
- Tasks written so they break into small steps
- Help initiating exercises in a focused way
- Can work independently through sets 1-2 once started

**Tier 3 — Intensive**: Severe learning differences (e.g., Nora-level dyslexia). Requires:
- Specialized scaffolding
- Constant monitoring
- Individualized learning path

**What MC would do**: Classify each student query/observation into tier. Route to appropriate support artifacts. Track progression across tiers over time.

## Use Case 3: Help Artifacts — Checklists and Timers

**The scenario**: A student who could spend an entire block sharpening a pencil. Cannot initiate work. Needs external structure.

**What the teacher builds (MC assists)**:
- Checklist with embedded timer
- Task 1: Just start (2 min)
- Task 2: Grab your materials — pencil + journal (2 min, with photos)
- Task 3: Write the date (2 min)
- Task 4: Write the title (2 min)
- Movement break after 4 tasks

**Key insight**: The student needs SHORTENED tasks they can actually complete. Something manageable. The checklist IS the scaffold — images, timers, micro-steps.

**What MC would do**: Generate checklist artifacts per student profile. Teacher says "this student needs initiation support for writing block" → MC produces a visual checklist template with timer structure. Teacher adjusts and prints.

## Use Case 4: Content Differentiation by Level

**The scenario**: Teaching Italian articles (il/la, nominativo). The standard textbook assumes Italy-level L1. Immersion students need the same concept at accessible level.

**What the teacher needs**: Take the content (articles) and find the right level. Already working with AI to create accessible versions — images, simplified text, visual vocabulary (il tavolo, la mamma).

**Broader application**: Plan a math lesson → create 3 groups at different levels. Same concept, different complexity. Distributed by lens.

**What MC would do**:
1. Teacher inputs: "Tomorrow: multiplication, 3 groups needed"
2. MC reads class profiles → clusters into 3 levels
3. MC generates: Level 1 (concrete/visual), Level 2 (standard), Level 3 (extension/challenge)
4. Teacher reviews and adjusts

## Use Case 5: Social-Emotional Awareness

**The scenario**: A student cannot work if seated near a student with whom they have conflict. Different tables needed.

**What the teacher needs**: Seating/grouping recommendations that account for social-emotional dynamics, not just academic level.

**What MC would do**: Student lenses include social-emotional notes. When generating group recommendations, MC flags known conflicts or productive pairings. "Note: Student A and Student B should not be in the same group this week (conflict logged 06/15)."

## Use Case 6: Weekly Routines and Correction

**The scenario**: Verbal conjugation routines — correcting students' routine errors week by week.

**What the teacher needs**: Track which conjugation patterns each student struggles with. Surface the right practice for the right student.

**What MC would do**: Lens trained on records. "Student X consistently errors on passato prossimo with essere. Suggested routine: 3 targeted exercises this week on past participle agreement."

## Use Case 7: Parent Recommendations (No AI Exposure)

**CRITICAL RULE**: Parents never see AI, never interact with lenses, never get AI-generated content directly.

**What the teacher DOES share with parents** (MC-suggested, teacher-approved):
- "Your child needs a creative quiet space at home"
- "A dedicated workspace helps focus"
- "Please read instructions more carefully with them"
- "Ask your child for examples when they explain their day"

**What MC would do**: Based on student lens observations, generate parent recommendation suggestions. Teacher reviews, personalizes, sends through normal channels (not AI-branded).

## Use Case 8: Student AI Exposure (Age-Appropriate)

### Elementary (Grade 3+, Chromebooks)
- First year: knowledge of what AI IS
- Non-critical-thinking tasks — fun, exploratory
- Structured tasks: "How many parts of an essay are there? What is a thesis statement?"
- Consequential thinking games — expose to AI through content-integrated experiences
- Exchange ideas, express ideas through AI-assisted exercises

### Middle School
- One project per year
- Check what is written by hand vs. AI-assisted (essays)
- Grade 8: harder project — learn HOW to use AI responsibly
  - Design a building, design a boat — creative + technical
  - The AI is a tool, the student is the architect
