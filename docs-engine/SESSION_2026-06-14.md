### Phase 2: Classification Engine Breakthrough

**Problem**: The ontology classifier misclassified 68% of queries because CORE intent nodes (CREATE, RESEARCH, DECIDE) hijacked domain-specific queries. Common English words like "write", "build", "search" fired as CORE signals and won over specific RIUs. The original dogfood query "Implement a governed Codex adapter" routed to CORE-CREATE instead of RIU-029.

**What we discovered**:
1. CORE nodes are INTENTS (what do you want to do). Specific RIUs are DOMAINS (what is this about). They're different axes jammed into one competitive scoring pass.
2. Every RIU already has a descriptive name ("Convergence Brief", "Tool-Calling Safety Envelope"). Users type those names. But the names weren't indexed as signals.

**What we built**:
- CORE-as-fallback: specific RIUs always win over generic CORE nodes (`engine.py:193-199`) (Claude)
- Name auto-indexing: every RIU's name is tokenized into the signal index (`engine.py:131-142`) (Claude)
- Description auto-indexing: node descriptions indexed with stricter threshold (`engine.py:144-152`) (Kiro)
- Relaxed threshold for name-derived signals: 2 matching tokens is enough (`engine.py:188-191`) (Claude)
- Description-derived signals: 3+ tokens required to reduce noise (`engine.py:190-191`) (Kiro)
- Manual signal enrichment: 11 governance signals on RIU-029, 6 boundary signals on RIU-514 (Claude)
- 4 golden governance test cases verified (Kiro)

**Result**: Classification accuracy 20% → 55% on 97-query golden dataset. 2.75x improvement from structural engine fixes, zero manual signal authoring needed for the improvement.

## Key Architectural Insights (Discovered During Build)

1. **CORE nodes are intents, not domains.** They should be fallbacks, not competitors. This one insight gave 35 percentage points of classification improvement.

2. **RIU names ARE the signals users type.** Auto-indexing node names and descriptions gave 2.75x accuracy improvement with zero manual work.

3. **One sanitizer service, not three implementations.** Pattern divergence between surfaces is a policy split waiting to happen. The unified service is built; wiring is next.

4. **Policy compliance must be verified, not trusted.** When an agent says "I didn't call external models," cross-reference with the firewall log. Trust but verify.

5. **PII protection must extend to disk.** Offline queue writes to IndexedDB must be stripped before persistence — "local-only" means nothing if sensitive data is on disk in plain text.

6. **Bridges must be governed by default.** Sanitization in the base class means no developer can accidentally skip it.

7. **Instance factory before features.** The pipeline is the product. The ontology is per-client content. Deploy story matters more than feature count.
