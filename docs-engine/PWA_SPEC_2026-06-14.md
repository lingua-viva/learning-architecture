## Overview

Transform the existing browser surface into an installable Progressive Web App.
Same codebase, same governance, same pipeline — but installable to home screen,
works offline for local operations, and feels native on every device.

**Principle**: The PWA IS the browser surface. Not a separate build. Adding PWA
capability means adding a manifest + service worker to what already exists.

## Service Worker (sw.js)

### Caching Strategy

| Resource | Strategy | Why |
|----------|----------|-----|
| `index.html` | Network-first, cache fallback | Always get latest UI |
| CSS/JS (inline in index.html) | Cache with HTML | Single-page, all inline |
| `logo-mc.png`, icons | Cache-first | Static assets, rarely change |
| `/api/chat` | Network-only + offline queue | Requires live backend |
| `/health` | Network-only | Real-time status check |
| `offline.html` | Pre-cached on install | Fallback when completely offline |

### Offline Queue (Critical for Still I Rise)

When the device is offline:
1. User types a query
2. SW intercepts the `/api/chat` POST
3. Stores the request in IndexedDB (`mc-offline-queue`)
4. Shows user: "📤 Queued — will send when connected"
5. When `online` event fires → SW replays queue in order
6. Each replayed response is delivered as a notification

```javascript
// Offline queue schema (IndexedDB)
{
  store: "mc-offline-queue",
  entries: [
    {
      id: "uuid",
      timestamp: "ISO8601",
      request: {
        agent: "perplexity",
        text: "What is the standard of care for...",
        mode: "research",
        system: "...",
        direct: false
      },
      status: "queued" | "sent" | "delivered",
      response: null | { ... }
    }
  ]
}
```

### Service Worker Lifecycle

```javascript
// sw.js core structure

const CACHE_NAME = 'mc-v1';
const OFFLINE_QUEUE_DB = 'mc-offline-queue';
const STATIC_ASSETS = ['/', '/logo-mc.png', '/offline.html'];

// INSTALL: Pre-cache static assets
self.addEventListener('install', (event) => {
  event.waitUntil(
    caches.open(CACHE_NAME)
      .then(cache => cache.addAll(STATIC_ASSETS))
      .then(() => self.skipWaiting())
  );
});

// ACTIVATE: Clean old caches
self.addEventListener('activate', (event) => {
  event.waitUntil(
    caches.keys().then(keys =>
      Promise.all(keys.filter(k => k !== CACHE_NAME).map(k => caches.delete(k)))
    ).then(() => self.clients.claim())
  );
});

// FETCH: Network-first for HTML, cache-first for assets, queue for API
self.addEventListener('fetch', (event) => {
  const url = new URL(event.request.url);

  // API calls: network-only, queue if offline
  if (url.pathname.startsWith('/api/')) {
    event.respondWith(handleApiRequest(event.request));
    return;
  }

  // Static assets: cache-first
  if (url.pathname.match(/\.(png|jpg|svg|ico)$/)) {
    event.respondWith(cacheFirst(event.request));
    return;
  }

  // HTML: network-first with offline fallback
  event.respondWith(networkFirstWithFallback(event.request));
});

// SYNC: Replay offline queue when back online
self.addEventListener('sync', (event) => {
  if (event.tag === 'mc-sync-queue') {
    event.waitUntil(replayOfflineQueue());
  }
});

// ONLINE: Also try to replay (belt + suspenders)
self.addEventListener('message', (event) => {
  if (event.data === 'online') {
    replayOfflineQueue();
  }
});
```

## index.html Changes

Add to `<head>`:
```html
<link rel="manifest" href="/manifest.json">
<meta name="theme-color" content="#111111">
<meta name="apple-mobile-web-app-capable" content="yes">
<meta name="apple-mobile-web-app-status-bar-style" content="black-translucent">
<meta name="apple-mobile-web-app-title" content="Mission Canvas">
<link rel="apple-touch-icon" href="/icons/icon-192.png">
```

Add before `</body>`:
```html
<script>
// Register service worker
if ('serviceWorker' in navigator) {
  navigator.serviceWorker.register('/sw.js')
    .then(reg => console.log('[MC] SW registered:', reg.scope))
    .catch(err => console.error('[MC] SW failed:', err));

  // Notify SW when back online
  window.addEventListener('online', () => {
    navigator.serviceWorker.controller?.postMessage('online');
  });
}
</script>
```

## Install Prompt Strategy

Don't show install prompt immediately. Show after:
1. User has submitted at least 2 queries (proven engagement)
2. User has been on the page for > 60 seconds
3. The `beforeinstallprompt` event fires (browser-native)

```javascript
let installPrompt;
window.addEventListener('beforeinstallprompt', (e) => {
  e.preventDefault();
  installPrompt = e;
  // Show custom install banner after engagement threshold
  if (queryCount >= 2) showInstallBanner();
});

function showInstallBanner() {
  // Subtle banner at top: "Install Mission Canvas for quick access →"
}
```

## Governance in PWA Context

**Critical**: The PWA must maintain full governance even when the underlying
connection model changes.

| Scenario | Behavior |
|----------|----------|
| Online, local MC installed | Hits `localhost:7890` — full pipeline |
| Online, using public site | Hits `srv1390882.hstgr.cloud:443` — hub pipeline |
| Offline, local MC installed | Local Ollama still works for PROTECT/REFLECT intents. RESEARCH queued. |
| Offline, public site only | ALL queries queued. Show offline.html for new sessions. |
| Reconnecting | Queue replays through sanitizer → pipeline → store. Governance fires on every replayed query. |

**The offline queue does NOT bypass governance.** Queries are stored raw locally,
then processed through the full pipeline when connectivity returns. The user sees
"queued" not "answered" — they never get an ungoverned response.
